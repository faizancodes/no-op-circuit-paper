Figure 8. Real-task generalisation of the FROZEN, toy-trained v_noop (no retraining) evaluated on paired prompts derived from SWE-bench_Verified. Same protocol as Figure 7; the residual at the intervention site (L24/pos −1 for Qwen, L26/pos −1 for CodeGemma) is projected onto its model's toy-trained unit vector and the resulting scalar is the classifier score. Qwen-1.5B: ROC-AUC = 0.989, AP = 0.991, precision = 0.97, recall = 0.94, projection gap = +4.73 over N = 499 paired tasks CodeGemma-7B: ROC-AUC = 0.933, AP = 0.927, precision = 0.85, recall = 0.89, projection gap = +2.86 over N = 497 paired tasks DeepSeek-1.3B: ROC-AUC = 0.888, AP = 0.879, precision = 0.81, recall = 0.83, projection gap = +5.64 over N = 499 paired tasks. Both models exhibit clean linear separability on real GitHub-issue prompts, confirming that the no-op direction is not a toy-distribution artefact.
